# Contributing to awesome-rocketride

## What belongs here

Public repos that include a working RocketRide pipeline or integration with a README explaining what it does. Demos, examples, and experiments all qualify. No broken links, no private repos, no purely promotional content.

## Entry format

Entries go under the appropriate section in `README.md`. Format is exactly:

```
- [Project Name](url) - One sentence description.
```

The description must say what the project does and which RocketRide feature or node type it uses. The sentence ends with a period.

## How to submit

1. Fork the repo.
2. Add your entry in alphabetical order within its section.
3. Open a PR. PR title format: `Add [Project Name]`.

PRs that touch anything outside `README.md` without a clear reason will not be merged.

## Criteria checklist

Before submitting, make sure all of the following are true:

- [ ] Repo is public
- [ ] Repo contains a working RocketRide pipeline JSON or integration code
- [ ] Repo has a README
- [ ] Entry follows the exact format above
- [ ] Link is not a duplicate

## Review process

Maintainers will review and leave feedback or merge within a reasonable time. No SLA guaranteed.
